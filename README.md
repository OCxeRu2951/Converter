# メディア変換・LUFS詳細解析・正規化ツール

Node.js と ffmpeg を活用し、高度な設定を対話形式で行えるメディア変換・解析ツール群です。

* **convert.js**: 動画・音声の形式変換、および `loudnorm` フィルタを用いたラウドネス正規化。
* **checker.js**: `ffprobe` による詳細なメタデータ確認と、EBU R128 規格に基づくラウドネス測定。

動作には **Node.js** と **ffmpeg / ffprobe** がシステムにインストールされている必要があります。

---

## 依存ライブラリ

本プロジェクトは `fluent-ffmpeg` を使用しています。以下のコマンドで依存関係をインストールしてください。

```bash
npm install
```

---

## 使い方

### 1. convert.js（詳細設定・正規化）

スクリプトを実行すると、ルートディレクトリ内のファイルを自動スキャンします。対象が見つからない場合は、ファイルパスを直接入力することも可能です。

1. **ファイル選択**: リストから番号を選択するか、パスを直接入力します。
2. **出力タイプ**: [1] 映像ファイル または [2] 音声ファイル を選択します。
3. **詳細エンコード設定**:

* **映像**: コーデック（H.264/H.265/ProRes/VP9）、プリセット、品質方式（CRF または Bitrate）を個別に指定。
* **音声**: コーデック（AAC/Opus/MP3/PCM/FLAC）、サンプルレート、ビットレート、チャンネル数を指定。

1. **正規化**: ターゲット LUFS（-14 固定）と True Peak（任意入力）に基づいて処理が実行されます。

**実行コマンド:**

```bash
node convert.js
```

---

### 2. checker.js（詳細解析）

指定したメディアファイルの技術情報を包括的に表示します。

1. **基本情報**: ファイル形式、再生時間、ファイルサイズ、全体のビットレート。
2. **オーディオ詳細**: コーデック（詳細名称）、サンプリングレート、チャンネル構成、ビットレート。
3. **ビデオ詳細**: 解像度、フレームレート。
4. **ラウドネス測定 (EBU R128)**:

* Integrated (I): 統合ラウドネス値
* True Peak (TP): トゥルーピーク
* Loudness Range (LRA): ラウドネスレンジ
* Threshold: スレッショルド
* Target Offset: ターゲットとのオフセット値

**実行コマンド:**

```bash
node checker.js
```

## 対応フォーマット

自動スキャンの対象となる主な拡張子：

* **音声**: `.mp3`, `.wav`, `.ogg`, `.flac`, `.aac`, `.opus`, `.m4a`, `.wma`, `.aiff`, `.alac`, `.dsf`, `.dff` 等
* **動画**: `.mp4`, `.mov`, `.mkv`, `.avi`, `.wmv`, `.flv`, `.webm`, `.ts` 等

---

## 主な機能と特徴

* **高度なパラメータ指定**: CRF値の直接入力や、Opus形式選択時の 48kHz 強制変換など、技術的な整合性を維持した変換が可能です。
* **詳細な解析表示**: `checker.js` では単なる LUFS だけでなく、ファイルに含まれる各ストリームの技術詳細を表示します。
* **パス入力の柔軟性**: カレントディレクトリにファイルがない場合でも、ドラッグ＆ドロップ等による絶対パス入力で処理を続行できます。
* **ストリーミング最適化**: MP4 や MOV 形式での出力時、自動的に `faststart` フラグを付与し、ウェブ上での再生開始を高速化します。

---

## 注意事項

* LUFS 測定・変換は ffmpeg の `loudnorm` フィルタ（2-pass 相当の処理）に依存しています。
* 変換後のファイルは、元のファイル名に `_processed` が付与され、ルートディレクトリに保存されます。
